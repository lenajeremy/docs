## Overview

Every endpoint defined with `builder.mutation` generates a `useXxxMutation` hook, where `Xxx` is the capitalised endpoint key.

```ts
// Endpoint key: createTodo  → Hook: useCreateTodoMutation
// Endpoint key: deletePost  → Hook: useDeletePostMutation
// Endpoint key: login       → Hook: useLoginMutation
```

Mutations are for writing data. Unlike queries, they are never cached and do not fetch automatically — you always call `trigger` explicitly.

---

## Signature

```ts
function useXxxMutation(
  options?: MutationHookOptions
): {
  data: Returns | undefined;
  loading: boolean;
  error: Error | undefined;
  trigger: (args: Takes) => Promise<Returns | undefined>;
}
```

---

## MutationHookOptions

<ParamField body="invalidates" type="TagType<TagString, Returns>">
  Tags to invalidate when this mutation succeeds — specified at the **call site** rather than at endpoint definition time. These are checked alongside any `invalidatesTags` set on the endpoint itself.

  Useful when the same mutation endpoint needs to bust different caches depending on where it is called.

  ```tsx
  const { trigger } = useDeletePostMutation();

  // Invalidates both the post list and the feed
  trigger({ id: 42 }, { invalidates: ['posts', 'feed'] });
  ```

  Supports all tag forms: plain strings, `{ name, id }` objects, and functions that receive the mutation response:

  ```tsx
  trigger(args, {
    invalidates: (res) => res ? [{ name: 'posts', id: res.id }] : [],
  });
  ```
</ParamField>

---

## Return value

<ResponseField name="data" type="Returns | undefined">
  The response data from the last successful `trigger` call. `undefined` before the first successful call.
</ResponseField>

<ResponseField name="loading" type="boolean">
  `true` while the mutation is in flight. Use this to disable submit buttons or show spinners.
</ResponseField>

<ResponseField name="error" type="Error | undefined">
  Set when the request throws or returns a non-2xx status. Cleared at the start of each new `trigger` call.
</ResponseField>

<ResponseField name="trigger" type="(args: Takes, options?: MutationHookOptions) => Promise<Returns | undefined>">
  Fire the mutation. Throws on failure, so wrap in `try/catch` for imperative error handling. Accepts an optional second argument to pass per-call `invalidates` tags.
</ResponseField>

---

## Error handling

`trigger` throws on failure. The `error` state is also set automatically. You can handle errors declaratively via the `error` field, imperatively via `try/catch`, or both:

```tsx
const { trigger, error, loading } = useCreateTodoMutation();

// Declarative: render error from state
{error && <p className="error">{error.message}</p>}

// Imperative: catch in the handler
async function handleSubmit() {
  try {
    const result = await trigger({ title });
    router.push('/todos');        // only runs on success
  } catch {
    // error state is already set — no need to do it manually
  }
}
```

## Cache invalidation on success

When `trigger` succeeds and the endpoint has `invalidatesTags` configured, quokkajs automatically re-fetches any currently-mounted queries whose `providesTags` overlaps.

```ts
deleteTodo: builder.mutation<{ id: number }, void>((args) => ({
  url: `/todos/${args.id}`,
  method: 'DELETE',
}), {
  invalidatesTags: ['todos'], // re-fetches useGetTodosQuery after delete
}),
```

See [Tag-based Invalidation](/concepts/invalidation) for the full details.

## Examples

### Form submission

```tsx
function LoginForm() {
  const [email, setEmail] = React.useState('');
  const { trigger, loading, error } = useLoginMutation();

  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault();
    try {
      const result = await trigger({ email });
      console.log('Logged in:', result);
    } catch {
      // error state set automatically
    }
  };

  return (
    <form onSubmit={handleSubmit}>
      <input
        type="email"
        value={email}
        onChange={(e) => setEmail(e.target.value)}
        placeholder="Email address"
      />
      <button type="submit" disabled={loading}>
        {loading ? 'Logging in...' : 'Log in'}
      </button>
      {error && <p style={{ color: 'tomato' }}>{error.message}</p>}
    </form>
  );
}
```

### CRUD with invalidation

```tsx
function TodoItem({ todo }: { todo: Todo }) {
  const { trigger: markDone, loading: marking } = useMarkAsDoneMutation();
  const { trigger: deleteTodo, loading: deleting } = useDeleteTodoMutation();

  return (
    <div>
      <input
        type="checkbox"
        checked={todo.isDone}
        onChange={() => markDone({ id: todo.id })}
        disabled={marking}
      />
      <span>{todo.title}</span>
      <button
        onClick={() => deleteTodo({ id: todo.id })}
        disabled={deleting}
        style={{ background: 'tomato' }}
      >
        {deleting ? 'Deleting...' : 'Delete'}
      </button>
    </div>
  );
}
```

### File upload with FormData

Mutations detect `FormData` automatically and send it as-is instead of JSON-encoding it:

```tsx
const { trigger: uploadAvatar } = useUploadAvatarMutation();

async function handleFileChange(e: React.ChangeEvent<HTMLInputElement>) {
  const file = e.target.files?.[0];
  if (!file) return;

  const formData = new FormData();
  formData.append('avatar', file);

  await uploadAvatar(formData);
}
```
