**quokkajs** lets you define your API endpoints once and get fully typed React hooks back automatically — with built-in caching, tag-based cache invalidation, debouncing, polling, and more.

Inspired by [RTK Query](https://redux-toolkit.js.org/rtk-query/overview), but without the Redux dependency.

```bash
npm install quokkajs
# or
yarn add quokkajs
```

---

## Why quokkajs?

Every React app that talks to an API ends up writing the same boilerplate: `useState` for data, loading, and error; `useEffect` to trigger the fetch; manual cache invalidation after mutations. quokkajs eliminates all of it.

You describe your endpoints. quokkajs generates the hooks.

```ts
// Define once
const todoApi = createApi({
  apiName: 'todoApi',
  baseUrl: 'https://api.example.com',
  endpoints: (builder) => ({
    getTodos: builder.query<void, Todo[]>(() => ({ url: '/todos' })),
    createTodo: builder.mutation<{ title: string }, Todo>((body) => ({
      url: '/todos',
      method: 'POST',
      body,
    })),
  }),
});

// Use anywhere
export const { useGetTodosQuery, useCreateTodoMutation } = todoApi.actions;
```

## Key Features

<CardGroup cols={2}>
  <Card title="Auto-generated Hooks" icon="wand-magic-sparkles">
    Define an endpoint key, get a `useXxxQuery` or `useXxxMutation` hook back automatically — fully typed, no boilerplate.
  </Card>
  <Card title="Built-in Cache" icon="database">
    Every query response is cached. Identical requests are served instantly without a network round-trip.
  </Card>
  <Card title="Tag-based Invalidation" icon="tags">
    Mutations automatically re-fetch the right queries on success. No manual `refetch()` calls needed.
  </Card>
  <Card title="TypeScript-first" icon="code">
    End-to-end type inference from endpoint definition to hook return value. Zero manual annotation required.
  </Card>
  <Card title="No Redux required" icon="circle-minus">
    Works standalone with any state management setup — Zustand, Redux, Context, or nothing at all.
  </Card>
  <Card title="Composable options" icon="sliders">
    Polling, debouncing, focus-refetch, and args-change reactivity — all configurable per hook call.
  </Card>
</CardGroup>
